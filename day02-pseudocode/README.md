# Day 02 — Pseudocode

## What I'm covering today
- Why pseudocode exists (language-agnostic planning)
- Standard conventions: INPUT, OUTPUT, IF/ELSE, WHILE, FOR
- Translating a flowchart into pseudocode
- Translating pseudocode into plain English and back

## Things to remember
- Pseudocode is not real syntax — it's structured plain English
- The goal is clarity of logic, not correctness of syntax
